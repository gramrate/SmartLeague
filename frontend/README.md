# SmartLeague Frontend

Frontend для SmartLeague на `React + TypeScript + Vite` с интеграцией в существующий backend API.

## Стек
- React 19
- TypeScript
- Vite
- TanStack Router
- TanStack Query
- Zustand (store авторизации)
- Cookie-based auth (через backend)

## Запуск

Требования:
- Node.js 20+
- Запущенный backend SmartLeague (по `backend/README.md`)

1. Установка зависимостей:
```bash
cd frontend
npm install
```

2. Настройка API (опционально):
```bash
# по умолчанию используется http://localhost:8000
export VITE_API_BASE_URL=http://localhost:8000
```

3. Dev-режим:
```bash
npm run dev
```

4. Production build:
```bash
npm run build
```

## Архитектура
- `src/lib/api.ts` — единый API-клиент и все вызовы backend
- `src/types/api.ts` — DTO/типы API
- `src/lib/auth-store.ts` — централизованное состояние `me` + auth bootstrap
- `src/routes/*` — страницы
- `src/components/site/*` — общие блоки UI

## Сессия и авторизация
- Backend использует cookie auth (`credentials: include`).
- После логина/регистрации загружается и хранится `me`.
- При `401` фронт делает `POST /api/v1/auth/refresh` и повторяет запрос.
- Публичные страницы доступны без регистрации.

## Иерархия ролей клуба
Используется строго:
- `0` — none
- `1` — member
- `2` — resident
- `3` — leader
- `4` — president

Человеко-читаемые лейблы отображаются в UI (`src/lib/roles.ts`).

## Реализованные страницы
- `/` — главная
- `/login`, `/register`
- `/clubs`, `/clubs/:id`, `/clubs/:id/manage`, `/clubs/:id/games`, `/clubs/:id/series`
- `/series`, `/series/create`, `/series/:id`
- `/game/:id`, `/game/:id/manage`
- `/players`, `/user/:id`
- `/account`

## Реализованные бизнес-правила

### Доступ к управлению клубом/сериями/играми
Права на действия проверяются по правилу:
- `me.club_id === currentClubId`
- `me.club_state in [3, 4]`

Применено к:
- кнопке/действию создания серии
- созданию игр в серии
- панели управления клубом
- действиям управления участниками/сериями/играми

### Президент в управлении участниками
- Президент вынесен отдельно в верхний блок с лейблом `Президент`.
- Президент исключен из управляемого списка участников.
- В UI нет механики случайного изменения роли президента через список.

### Ограничения по клубам
- Если пользователь уже в клубе, на странице клубов не показывается кнопка `Создать клуб`.
- Если пользователь уже в клубе, на странице другого клуба кнопка вступления не показывается, выводится понятное сообщение.

## Разделы

### Clubs
- Публичный список клубов.
- Детальная клуба: описание, участники, серии, текущие игры.
- Переходы на полный список игр и серий клуба.
- Управление (leader/president своего клуба):
  - редактирование клуба
  - удаление серий
  - удаление игр
  - управление участниками: смена роли, kick, block, передача президентства

### Series
- Страница всех серий использует `GET /api/v1/series/all`.
- Фильтры: название, клуб, даты, наличие игр.
- Страница серии: данные, участники, leaderboard, игры, переход в клуб.
- Для manager клуба: форма создания игры.
- В создании серии — поле `description`, без `rules/scoring rules/status` в UI.

### Games
- Страница игры показывает участников и результаты.
- В UI отображаются никнеймы/имена, а не UUID.
- В таблице результатов: сначала место, затем имя.
- Переход к серии игры.
- Номер игры вручную не вводится (назначается backend).

### Players (публично)
- Вкладка Players доступна без регистрации.
- Поиск игроков по общему запросу (включая никнейм).
- Профиль игрока: базовая информация, клуб с переходом, последние 3 игры, показать все игры, показать все серии.

## Состояния UI
Во всех ключевых страницах есть:
- loading
- empty
- error

Backend-ошибки выводятся в тостах/блоках ошибок.

## API и Swagger
Frontend опирается на backend API из Swagger (`backend/docs/swagger.json`).
Ключевые интеграции:
- `GET /api/v1/series/all` для общего списка серий
- `GET /api/v1/series/{id}/full` для агрегированной страницы серии

## Отчет по правам доступа
- `none/member/resident`:
  - просмотр публичных страниц
  - нет доступа к управлению клубом/сериями/играми
- `leader` своего клуба:
  - доступ к панели управления клубом
  - создание серий
  - создание игр в сериях своего клуба
  - управление участниками (ограничено backend-правилами)
- `president` своего клуба:
  - полный доступ к управлению клубом
  - отдельный блок президента в UI

Важно: те же роли в другом клубе не дают доступ к управлению текущим клубом.

## Известные ограничения
- Поиск игроков использует backend-поиск `GET /api/v1/user/all`; точность поиска по никнейму зависит от серверной реализации фильтра `q`.
- Для детальной фильтрации/сортировок страниц клуба используются текущие публичные API backend без дополнительных специализированных агрегирующих ручек.
